01 · Overview

What is PRAJNA?

Moving beyond guesswork to data-driven prediction at micro-topic granularity

🎯

What it Does

PRAJNA ingests 48 years of JEE and NEET question papers and applies a 3-stage ML pipeline to predict which topics, micro-topics, and question types are most likely to appear in the next exam — with subject-balanced output.

🧩

Problem it Solves

Students waste time on low-yield chapters while missing high-probability micro-topics. Coaching institutes rely on intuition and hearsay. PRAJNA replaces guesswork with statistical signal backed by nearly five decades of real exam data.

⚡

Why it Matters

JEE and NEET are fiercely competitive. A single correctly predicted micro-topic can swing a rank by thousands. PRAJNA gives every student access to institutional-level prediction intelligence — for free.

Most students study broad chapters. PRAJNA tells you EXACTLY which micro-topics to focus on — backed by 48 years of actual exam data. It goes beyond chapter-level analysis all the way down to micro-topic level predictions: 470 micro-topics for NEET and 285 for JEE, each with an individual probability score, expected question count, and confidence rating.

384-Dimensional Semantic Embeddings
Chapter names are encoded using sentence-transformers/all-MiniLM-L6-v2, giving the SLM semantic understanding of topic relationships. Chapters with similar names (e.g., "Laws of Motion" and "Newton's Laws") share representation space — enabling cross-chapter pattern transfer not possible with heuristics alone.

Custom Model or Wrapper?

🔒 Pretrained & Frozen
sentence-transformers/all-MiniLM-L6-v2
Used only as a text encoder to convert chapter names → 384-dim vectors. Weights are frozen during training — PRAJNA never fine-tunes this model.
✅ Trained From Scratch
MLP backbone + 3 output heads
Trained entirely on 48 years of JEE/NEET exam data with strict temporal splits. No existing exam-prediction model was used or wrapped.
Analogy: Just as a vision model uses a pretrained ResNet as a feature extractor and trains a custom classification head, PRAJNA uses sentence-transformers for semantic features — but the actual predictor is 100% custom, trained on your domain-specific exam history.

Heuristic v3 — End-to-End Pipeline

Pure signal engineering, no gradient descent. Deterministic and fully interpretable.

1
Data Loading + Holdout Exclusion
Load all 23,119 questions from SQLite. Exclude holdout years {2024, 2025, 2026}. Filter by exam if specified. Group by (subject, chapter).
2
Syllabus Gate
Check each chapter against 2024 syllabus changes. REMOVED → gate=0.0 (score floored to 0). NEWLY ADDED → gate=0.5. RETAINED → gate=1.0.
3
Stage 1 — Appearance Probability (7-signal weighted ensemble)
① Recency-Weighted Freq ×0.25: Σ 1/(gap^1.5+1) normalized over full year range
② Appearance Rate ×0.20: count_years_appeared / year_span
③ Recent 3yr ×0.15: appearances in last 3 exams / 3
④ Gap Return ×0.15: gap/mean_gap capped at 2, peaks when 2× overdue
⑤ Trend Slope ×0.12: np.polyfit on last 10 years of frequency
⑥ Cycle Match ×0.08: modular arithmetic on inter-gap variance (σ² ≤ 1.5)
⑦ Recent 5yr ×0.05: appearances in last 5 exams / 5
Result: min(weighted_sum × 1.8, 0.99) × syllabus_gate
4
Stage 2 — Expected Questions (Weightage Model)
Blend: 50% recency-weighted avg (decay 1/√gap) + 30% recent 5yr avg + 20% overall median. Confidence = 1 − coefficient_of_variation. Range = [Q25, Q75].
5
Stage 3 — Format Prediction + Final Scoring
Format: top-3 historical question types by frequency. Difficulty: recency-weighted mean of last 5 years.
Final = 0.40×P(appear) + 0.30×norm_qs + 0.15×yield_bonus + 0.10×cross_exam + 0.05×syl_gate
Subject-balanced reranking: NEET → Bio 50% / Phy 25% / Chem 25% · JEE → equal thirds. Chapter deduplication via alias map.

PRAJNA SLM — End-to-End Training Pipeline

Data → Features → Embeddings → Multi-task training → Best checkpoint → Inference

1
Training Data Construction (2010–2023)
For each (topic, target_year) pair: extract 15 features using only data before target_year. Labels from the actual paper that year (appeared=1/0, num_qs, difficulty_class). Class ratio ~15:1 (negative:positive) → corrected via pos_weight = neg_count / pos_count.
2
Semantic Embedding (Frozen Backbone)
All topic names → all-MiniLM-L6-v2 → 384-dim vectors (batch_size=64). Cached to *_embeddings.npz after first run. Concatenated with 15 numerical features → 399-dim input tensor.
3
Model Architecture (PrajnaSLM)
Backbone: 399→256→128→64, each layer = Linear + LayerNorm + GELU + Dropout(0.2)
Appear head: 64→16→GELU→1→Sigmoid → P(appear) ∈ [0,1]
Questions head: 64→16→GELU→1→ReLU → expected count ≥ 0
Difficulty head: 64→16→GELU→3 → logits, softmax at inference → Easy/Med/Hard
4
Multi-Task Loss + Optimization
Loss = 0.50 × BCE(appear, pos_weighted) + 0.35 × MSE(qs | appeared=1) + 0.15 × CrossEntropy(difficulty)
MSE is masked — computed only on positive samples, preventing zero-inflation from non-appearing chapters.
Optimizer: AdamW lr=1e-3, weight_decay=0.01  ·  Scheduler: CosineAnnealingLR eta_min=1e-5
Gradient clip = 1.0  ·  150 epochs, batch_size=128  ·  Best checkpoint: NEET epoch 126, loss 0.8702
5
Inference + Final Scoring
Load best checkpoint, set model to inference mode, run with no-gradient context.
SLM Score = 0.45×P(appear) + 0.35×norm_exp_qs + 0.10×yield_bonus + 0.05×cross_exam + 0.05×syl_gate
Apply same subject-balanced reranking as heuristic v3. Embedding cache loaded from npz — avoids re-encoding at inference.
Heuristic v3 — Key Strength
Fully interpretable — every signal and weight is human-defined. Great when you need to justify a prediction to a student or teacher. No training required; works with as few as 5 historical papers.
PRAJNA SLM — Key Strength
Learns feature interactions automatically. Semantic embeddings generalize across similar topics. Masked MSE teaches how heavy a chapter will be — not just if it appears — driving the +4.2% coverage gain.

Snippet 1 · Appearance Probability

def _appearance_probability(
    years_appeared, total_years_range,
    target_year, max_year, decay=1.5
):
    # Signal 1: Recency-weighted frequency (exponential decay)
    rwf = sum(
        1.0 / ((target_year - y) ** decay + 1)
        for y in years_appeared
    )
    max_possible = sum(
        1.0 / ((target_year - y) ** decay + 1)
        for y in range(total_years_range[0], total_years_range[1] + 1)
    )
    rwf_norm = min(rwf / (max_possible + 0.01), 1.0)

    # Signal 4: Gap return probability
    gap = target_year - max(years_appeared)
    mean_gap = np.mean(inter_gaps) if inter_gaps else year_span
    gap_return = min(gap / (mean_gap + 0.1), 2.0) / 2.0

    # Weighted combination of all 7 signals
    score = (
        0.25*rwf_norm       +
        0.20*appearance_rate  +
        0.15*recent_3yr       +
        0.15*gap_return       +
        0.12*trend_slope      +
        0.08*cycle_match      +
        0.05*recent_5yr
    )
    return min(score * 1.8, 0.99), signals
Snippet 2 · Subject-Balanced Reranking
Python 
SUBJECT_QUOTAS = {
    "NEET":     {"Biology": 0.50, "Physics": 0.25, "Chemistry": 0.25},
    "JEE Main": {"Physics": 0.33, "Chemistry": 0.33, "Mathematics": 0.34},
}

def _subject_balanced_rerank(predictions, exam, top_k=50):
    quotas = SUBJECT_QUOTAS.get(exam, {})
    slots  = {
        s: max(3, int(top_k * q))
        for s, q in quotas.items()
    }
    # Fill per-subject buckets with highest-scoring topics
    result = []
    for s, slot_count in slots.items():
        subj_preds = [
            p for p in predictions
            if p["subject"] == s
        ]
        result.extend(subj_preds[:slot_count])
    result.sort(key=lambda x: x["final_score"], reverse=True)
    return result[:top_k]

08 · Ask PRAJNA

Exam Intelligence Chatbot

RAG-style conversational interface over 23,119 questions — ask anything about exam trends, topic difficulty, or historical patterns

🎯 6 Intent Types — Auto-Detected
  • 📊
    Count / Frequency
    Keywords: "how many", "count", "total", "number of"
    "How many times has Thermodynamics appeared in NEET?"
  • 📈
    Trend Analysis
    Keywords: "trend", "rising", "declining", "hot", "cold", "popular"
    "Is Organic Chemistry getting more important in JEE Main?"
  • 🎯
    Difficulty Analysis
    Keywords: "difficult", "hardest", "easiest", "hard", "easy", "tough"
    "What are the hardest Physics topics in JEE Advanced?"
  • ⏰
    Gap / Overdue
    Keywords: "gap", "overdue", "missing", "not appeared", "absent"
    "Which chapters are overdue in NEET 2026?"
  • ⚖️
    Subject Comparison
    Keywords: "compare", "vs", "versus", "comparison", "difference"
    "Compare Physics vs Chemistry difficulty in NEET"
  • 🔍
    Topic Detail + Semantic Search
    Detected topic in query → handler · otherwise → ST cosine-sim fallback
    "Tell me everything about Thermodynamics in JEE Main"

How many times has Human Physiology appeared in NEET?

Human Physiology has appeared in 42 of 48 NEET papers (87.5% rate). It averaged 11.3 questions per year and is the single highest-weighted chapter. Last appeared: 2025.
U
Which Physics topic is most likely due for a comeback in JEE 2026?
P
Based on gap analysis: Semiconductor Devices (last seen 2022, mean gap 2.1y), Communication Systems (last 2021, mean gap 1.8y), and Magnetism & Matter (last 2020) are the top 3 "overdue" chapters.
How it works: Each query is passed through an intent classifier. If semantic search is needed, the question is embedded via sentence-transformers and matched against all 23,119 question embeddings using cosine similarity. Results are re-ranked by recency and exam relevance.
